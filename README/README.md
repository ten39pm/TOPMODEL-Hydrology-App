# HBEF Watershed 3 — TOPMODEL Explorer

A Shiny web application for running TOPMODEL, a rainfall-runoff model that predicts streamflow based on watershed topography, soil properties, and climate inputs. Built for the **Hubbard Brook Experimental Forest (HBEF) Watershed 3** as part of the EDS Capstone project.

## Overview

This app simulates streamflow for the HBEF Watershed 3 using TOPMODEL, a topography-driven hydrological model. It allows users to:

- Manually tune model parameters via interactive sliders
- Automatically calibrate parameters using the Nelder-Mead optimizer
- Visualize simulated vs. observed discharge, flow duration curves, snow water equivalent, and ET
- Explore an animated **saturation map** showing which parts of the watershed are predicted to be saturated on any given day
- View the Topographic Wetness Index (TWI) map with elevation contours

## Before Running the App: Precompute TWI

**This step is required and must be completed before launching the app for the first time.**

The app depends on a precomputed Topographic Wetness Index (TWI) raster file (`Data/twi_precomputed.tif`). This file is generated by running WhiteboxTools (WBT) on the raw DEM — a process that is too slow to run every time the app starts, and is not available in deployed environments like shinyapps.io.

### Why this is necessary

When the app launches, it checks for `Data/twi_precomputed.tif`:

- **File exists** → the app loads it instantly and runs normally
- **File is missing** → the app will throw an error and refuse to start

### How to run it

Open `precompute_twi.R` in RStudio and click **Source**, or run from the R console:

```r
source("precompute_twi.R")
```

The script will print progress messages as it works through each step:

```
Loading DEM and watershed ...
Breaching depressions ...
FD8 flow accumulation ...
Computing slope ...
Computing TWI ...
Done! Saved to: Data/twi_precomputed.tif
You can now deploy to shinyapps.io without WhiteBox.
```

This only needs to be run **once**. After `twi_precomputed.tif` exists in the `Data/` folder, you can launch the app repeatedly without re-running it.

### What the script does

1. Loads the 10m DEM and clips it to the Watershed 3 boundary
2. Breaches depressions in the DEM (fills sinks) using WBT
3. Computes FD8 flow accumulation (specific contributing area)
4. Computes slope in degrees
5. Calculates TWI as `log(SCA / tan(slope))` and masks it to the watershed
6. Saves the result to `Data/twi_precomputed.tif`

## Running the App

Once `Data/twi_precomputed.tif` exists, launch the app from RStudio or the R console:

```r
shiny::runApp("app.R")
```

Or open `app.R` in RStudio and click **Run App**.

## Using the App

### Sidebar Controls

- **Viewing Period** — Select a date range (year slider). The model runs only on the selected window.
- **Run Model** — Runs TOPMODEL with the current slider parameters and updates all plots.
- **Optimize Parameters** — Automatically calibrates all 8 parameters using the Nelder-Mead algorithm against the selected date window. Tries 3 starting points and keeps the best result. Updates the sliders and re-runs the model.
- **NSE Score Box** — Displays the Nash-Sutcliffe Efficiency after each run. Green (≥ 0.5) is good, yellow (≥ 0) is acceptable, red (< 0) indicates a poor fit.

### Parameter Groups

| Group | Parameters | Controls |
|---|---|---|
| Subsurface Flow | `qs0`, `ln(Te)`, `m`, `td` | Baseflow magnitude and drainage speed |
| Root Zone & ET | `Sr0`, `Srmax` | Soil moisture capacity and evapotranspiration |
| Snow | `Tmelt`, `Melt coeff` | Snow accumulation and degree-day melt |

### Plot Tabs

| Tab | Description |
|---|---|
| **Model Output** | 7-panel stacked time series: discharge, surface runoff, subsurface flow, ET, SWE, storage deficit, and saturated fraction |
| **Observed vs Simulated Discharge** | Scatter plot with 1:1 reference line |
| **Flow Duration** | Log-scale exceedance curve for observed and simulated flow |
| **TWI Map** | Static map of Topographic Wetness Index with elevation contours |
| **Saturation Map** | Animated daily map showing saturated vs. unsaturated cells alongside discharge and saturated area time series |

## Data Requirements

All input files should be placed in the `Data/` directory:

| File | Description |
|---|---|
| `dem_10m.tif` | 10-meter resolution Digital Elevation Model for HBEF W3 |
| `Watershed3HB.shp` | Watershed 3 boundary polygon (+ associated `.dbf`, `.prj`, `.shx`) |
| `DailyWatershed.csv` | Daily precipitation — must include `DATE` and `watershed` columns |
| `HBEF_DailyStreamflow_1956-2024.csv` | Daily streamflow — must include `DATE`, `WS`, and `Streamflow` columns |
| `HBEF_air_temp_daily.csv` | Daily air temperature — must include `date` and `AVE` columns |
| `twi_precomputed.tif` | **Generated by `precompute_twi.R`** — do not create manually |
